# Contributing Guide

Thank you for your interest in contributing to our projects!

Contributions must align with our goal: a reliable, high-performance and secure server for Lineage II, written in Go.

## How to Contribute

1. **Fork the repository** and create a feature branch.
2. **Write clear, tested, and well-documented code.** Follow idiomatic Go style.
3. **Update or add tests** as necessary to cover your changes.
4. **Document your changes** clearly in code comments and, if relevant, in the `README.md`.
5. **Open a pull request** with a clear explanation of the problem and solution.

## Reporting Bugs

- Open a [GitHub Issue](../../issues).
- Provide clear steps to reproduce the bug, observed and expected behavior, and relevant logs or error messages.

## Suggesting Enhancements

- File an issue to discuss significant feature proposals before you start coding.
- Please include motivation, alternatives considered, and potential impact.

## Code Style & Standards

- Format your code with `gofmt` and lint with `golangci-lint`.
- Adhere to idiomatic Go and project best practices.
- All code must remain compatible with the project's Go version (see `README.md`).

## Code of Conduct

Please read our [Code of Conduct](./CODE_OF_CONDUCT.md) before participating.
